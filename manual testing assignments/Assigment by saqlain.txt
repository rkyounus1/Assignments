Manual Testing Interview Preparation 

1) Manual Testing Roadmap
- Learn Software Basics
- Understand SDLC
- Learn STLC
- Learn Types of Testing
- Practice Test Cases
- Learn Bug Reporting
- Practice on real applications

2) Important Interview Questions

What is Manual Testing?
Manual Testing is the process of testing software manually without using script or automation tools.
What is SDLC?
SDLC is a step-by-step method used to create software in an organized way, including planning, building, testing, and maintaining it.

What is STLC?
STLC is a step-by-step process used by testers to check software quality by performing different testing stages.

What is Test Case?
A Test Case is a written document that includes steps to test, input values, expected output, and the actual result obtained after testing.

What is Bug Life Cycle?
It is the lifecycle of a defect from identification to closure.

3) Types of Testing

Functional Testing:
- Unit Testing: Testing individual components (done by developers)
- Integration Testing: Testing interaction between modules
- System Testing: Testing complete system
- Smoke Testing: Basic build verification
- Sanity Testing: Quick check of specific functionality
- Regression Testing: Ensures old features work after changes

Non-Functional Testing:
- Performance Testing: Checks system speed, responsiveness, and stability
- Load Testing: Checks system behavior under expected user load
- Stress Testing: Checks system under extreme conditions
- Security Testing: Ensures system is safe from threats
- Usability Testing: Checks user-friendliness
Important Testing Types (Simple)
Unit Testing
→ Done by developers, tests small parts of code.
Smoke Testing
→ Basic test to check if build is stable.
Sanity Testing
→ Check specific functionality after small changes.
Regression Testing
→ Check if old features still work after changes.

4) SDLC Phases (Software Development Life Cycle)

- Requirement Analysis: Understanding what needs to be tested
 Test Planning: Deciding how to test, what to test, and needed resources
 Test Case Writing: Preparing test scenarios and steps
 Test Execution: Running the test cases
 Bug Reporting: Identifying and reporting defects
 Test Closure: Completing testing and documenting results

5) STLC Phases (Software Testing Life Cycle)

- Requirement Analysis: Understanding testing requirements
- Test Planning: Defining strategy, scope, and resources
- Test Case Writing: Creating test cases
- Test Execution: Executing test cases
- Bug Reporting: Logging defects
- Test Closure: Finalizing testing process
Includes:

Smoke Testing
Sanity Testing
Regression Testing

6) Bug Life Cycle

- New: Bug is identified
- Assigned: Assigned to developer
- Open: Developer starts fixing
- Fixed: Bug is resolved
- Retest: Tester verifies fix
- Verified: Bug is confirmed fixed
- Closed: Bug is closed
(If not fixed → Reopened)

7) Severity vs Priority
Severity → How serious the bug is
(Example: App crash = High severity)
Priority → How fast it should be fixed
(Example: Login issue = High priority)

Severity: Impact of the defect
Priority: Urgency to fix the defect

8) Day-to-Day Activities of Tester

- Understand requirements
- Write test cases
- Execute test cases
- Identify bugs
- Report bugs
- Retest fixed bugs
- Attend meetings
- Perform regression testing
